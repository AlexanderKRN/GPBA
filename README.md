# GPBA - backend

#### КОММЕНТАРИИ :
#### * уделено внимание соблюдению принципов SOLID, а также пунктов конвенции по разработке C# Coding Conventions Microsoft
#### * применены принципы "Чистой архитектуры" c элементами DDD, в частности операции с сущностью "оффер" происходят через сущность "поставщика"
#### * БД - подход "Code first" посредством конфигурации Fluent API в EF Core, ручные миграции
#### * оптимизация запросов к БД :
####     > метод AsNoTracking для ускорения операций чтения (кэширование и пагинация не добавлены на данном этапе)
####     > ограничение размера полей таблиц nvarchar(50) для оптимизации размера БД
####     > создан индекс для таблицы офферов на поле "модель", как наиболее селективное 
####     > использован Dapper для прямого SQL-запроса при выборке топ-3 поставщиков
#### * применён "Result-паттерн" при обработке ошибок и Envelope для оформления ответа на UI
#### * валидация входящих данных : на уровне доменной логики и уровне конфигураций EF Core
#### * применены паттерны "Репозиторий" и "Unit of work" для обработки логики сущности поставщика
#### * документация в Swagger
